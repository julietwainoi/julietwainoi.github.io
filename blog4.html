Day 4: Deploying a Highly Available Web App on AWS Using Terraform

Today’s challenge was all about deploying a highly available, configurable web application on AWS using Terraform. Here's a summary of what I accomplished:

📘 Reading

I read Chapter 2 of "Terraform: Up & Running" by Yevgeniy Brikman, covering core concepts such as modules, state, and infrastructure design patterns.

🛠️ Hands-on Labs

  • Lab 05: Data Sources – Learned how to dynamically fetch data using Terraform's data blocks.
  • Lab 06: Variables – Practiced parameterizing infrastructure using variables.tf and terraform.tfvars.

🚀 Deployments

  • Configurable Web Server: Used variables to allow dynamic configuration of EC2 instance count, type, AMI, and key pair.
  • Clustered Web Server: Launched multiple instances across different AZs to simulate high availability.

📚 Learnings

I explored Terraform documentation to deepen my understanding of providers, resource blocks, and the Terraform workflow (init → plan → apply).

✅ Conclusion

This was a valuable step in my Infrastructure as Code journey. I now better understand how to manage scalable infrastructure declaratively and repeatably with Terraform.
