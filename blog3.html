🧠 Introduction:

In my recent Terraform learning journey, I explored some foundational concepts that are critical to managing infrastructure as code. In this post, I’ll walk you through Labs 03, 04, and 05, where I learned about providers, resource blocks, and data sources — all using Amazon Web Services (AWS).

🔍 Lab 03: Terraform Providers

A provider tells Terraform how to interact with a specific cloud platform or service. For this lab, I used the AWS provider.

Here’s how I declared it:


terraform {
  required_version = ">= 1.0.0"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 3.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}

This ensures that Terraform uses a compatible version of AWS provider in the us-east-1 region.
